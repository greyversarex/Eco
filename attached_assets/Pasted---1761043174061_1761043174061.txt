Финальное и Полное Техническое Задание для Платформы "ЭкоТочикистон"
Задача: Создать веб-приложение в строгом соответствии с 12 пунктами, перечисленными ниже. Этот план является окончательным и включает ключевые уточнения, которые имеют высший приоритет.
1. Панель Администратора:
Реализовать защищенную панель администратора по адресу /admin с аутентификацией по логину и паролю. Ее назначение — управление отделами и полный надзор за сообщениями. "Мониторинг системы" конкретно означает возможность администратора видеть все сообщения, которыми обмениваются все отделы.
2. Доступ для Отделов:
Основной способ доступа для отделов должен быть только через уникальные коды доступа. Для отделов не должно быть никаких индивидуальных учетных записей, логинов или паролей.
3. Управление Отделами:
Панель администратора должна позволять создавать, редактировать и удалять отделы. Каждому отделу должна быть присвоена визуальная группа ("Верхний", "Средний" или "Нижний" блок).
4. Система Сообщений:
Реализовать полноценную систему обмена сообщениями с отдельными разделами "Входящие" и "Исходящие". Непрочитанные сообщения должны быть четко выделены. Критически важно реализовать ответы на сообщения (threading), где ответы привязываются к исходному сообщению, формируя понятную историю переписки.
5. Форма Создания Сообщения:
Форма для создания нового сообщения должна включать обязательные поля для темы/типа документа, даты, получателя (выпадающий список всех отделов) и основного текстового поля. Она также должна включать необязательное текстовое поле для "Имени исполнителя".
6. Прикрепление Файлов:
Система должна позволять прикреплять файлы к сообщениям. Эти файлы в обязательном порядке должны загружаться во внешнее S3-совместимое облачное хранилище, и только ссылка на файл должна сохраняться в базе данных.
7. Главный Экран Отдела (ЭТО НЕ ДАШБОРД):
После входа пользователь отдела видит главный экран. Этот экран должен быть простым меню, а не аналитическим дашбордом. На нем должен отображаться список всех других отделов, визуально сгруппированных по блокам ("Верхний", "Средний", "Нижний"). У каждого отдела в списке должен быть видимый счетчик непрочитанных сообщений. На этом экране не должно быть никаких других виджетов, графиков или статистики.
8. Взаимодействие с Сообщениями:
Пользователи должны иметь возможность открыть любое сообщение в своих входящих, просмотреть его содержимое, скачать прикрепленный файл и использовать кнопку "Ответить", которая открывает форму для продолжения цепочки переписки.
9. Двуязычная Поддержка:
Весь интерфейс должен поддерживать два языка: Таджикский (по умолчанию) и Русский. Простой переключатель языков должен присутствовать и быть легко доступным.
10. Дизайн Пользовательского Интерфейса (UI):
Пользовательский интерфейс должен быть минималистичным и чистым. Используй белый или очень светло-серый фон. Зеленый должен быть единственным акцентным цветом, используемым для кнопок, ссылок и важных индикаторов. Избегай сложных теней, градиентов и разноцветных тегов.
11. База Данных:
Приложение в обязательном порядке должно использовать внешнюю базу данных PostgreSQL (например, через Neon или Supabase) для хранения всех данных, чтобы обеспечить их сохранность и целостность.
12. Архитектура Бэкенда:
Бэкенд в обязательном порядке должен быть построен как RESTful API. Это не подлежащее обсуждению требование, чтобы обеспечить будущую совместимость с мобильными приложениями для iOS и Android.
