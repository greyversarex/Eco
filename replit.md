# EcoDoc Project Progress

## Разделение типов документов и задач (Намуди ҳуҷҷат ва Намуди супоришҳо)
- **Проблема**: При добавлении типа документа он мог появляться в списке типов задач. Также на продакшене системные типы задач могли "перехватывать" категорию `message`, если пользователь создавал тип документа с таким же именем.
- **Причина**: 
  1. Недостаточная фильтрация на фронтенде.
  2. SQL-запрос `ON CONFLICT (name) DO UPDATE SET category = 'assignment'` в `server/index.ts` принудительно менял категорию на `assignment` для любого типа с системным именем, даже если пользователь создал его как `message`.
- **Решение**:
  1. **Бэкенд (`server/index.ts`)**: SQL-запрос обновления системных типов теперь содержит условие `WHERE category != 'assignment'`. Это предотвращает случайное переименование пользовательских типов документов в типы задач.
  2. **Бэкенд (`server/routes.ts`)**: Маршруты создания (`POST`) и обновления (`PATCH`) теперь явно обрабатывают поле `category`.
  3. **Фронтенд (`AdminDocumentTypes.tsx`)**: Введен строгий фильтр `dt.category === 'message'` и принудительная передача категории при сохранении.
  4. **Фронтенд (`AdminAssignmentTypes.tsx`)**: Введен строгий фильтр `dt.category === 'assignment'`.
  5. **База данных**: Проведено обновление всех существующих записей.

## Текущее состояние
- Приложение работает на порту 5000.
- Админ-панель: `admin` / `admin123`.
- Типы документов и типы задач теперь полностью изолированы. Системный механизм сидинга больше не портит пользовательские данные.
